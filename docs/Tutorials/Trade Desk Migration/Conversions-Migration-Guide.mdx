# Migrate a Conversion

## Overview

Trade Desk refers to their conversions as tracking tags. They use two types of conversions, similar to IQM. Their HTML conversion is called a "static tag" rather than IQM's "pixel"; their server-to-server conversion is a "universal pixel tag" rather than IQM's "postback". This page foregrounds IQM's usage. 

IQM's [Conversions API](/docs/Guidelines/Conversion-API-Guidelines) manages pixel and postback conversions. See the [Tracking Tags<ExternalLink />](https://partner.thetradedesk.com/v3/portal/data/doc/TrackingTagsOverview) page for Trade Desk's platform.

The table below highlights the main actions and API methods for managing conversions. Each IQM endpoint is paired with the analogous Trade Desk API. Both link to their respective documentation. By following the workflow instructions and comparing APIs, the user can migrate their data from Trade Desk to IQM.
