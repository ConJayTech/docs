# API Pagination

## Overview

IQM's API supports filtering and sorting in many of its endpoints. By either passing query or request parameters, results can be filtered for certain keywords or entity IDs, or, sorted to a specified order. 

Filtering and sorting parameters 

| Query Parameters |  |
| ---- | --- |
| `ids` <br /><span class="type-text">integer</span> | Filter results by entity ID |
| `searchField` <br /><span class="type-text">string</span> | Search results by keyword |
| `limit` / `noOfEntries` <br /><span class="type-text">integer</span> | Maximum number of entries returned |
| `pageNo` <br /><span class="type-text">integer</span> | Page number for the data |
| `sortBy` <br /><span class="type-text">string</span> | Sorts by ascending (<var>+</var>) or descending (<var>-</var>) |
