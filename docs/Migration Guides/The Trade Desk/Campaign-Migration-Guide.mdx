# Migrate a Campaign

## Overview

As outlined in the [Overview](/docs/Migration%20Guides/The%20Trade%20Desk/Overview) page, The Trade Desk refers to their Campaigns as Ad Groups; and their Insertion Orders as Campaigns. This page foregrounds IQM's usage.

IQM's [Campaign API](/docs/Guidelines/Campaign-API-Guidelines) manages Campaigns and Insertion Orders. See the [Campaigns<ExternalLink />](https://partner.thetradedesk.com/v3/portal/api/doc/Campaigns) page for The Trade Desk's platform.  

The table below highlights the main actions and API methods for creating a Campaign. Each IQM endpoint is paired with the analogous API for The Trade Desk. Both link to their respective documentation. By following the workflow instructions and comparing APIs, the user can migrate their data from The Trade Desk to IQM. 

 For a detailed breakdown of the Campaign creation workflow see the [Create a Campaign](/docs/Quickstart%20Guides/Upload-Creative-and-Create-a-Campaign-API-Quickstart-Guide) quickstart guide. See the [Campaign Creation Workflow with REST<ExternalLink />](https://partner.thetradedesk.com/v3/portal/api/doc/CampaignCreateREST) page on The Trade Desk's website.
