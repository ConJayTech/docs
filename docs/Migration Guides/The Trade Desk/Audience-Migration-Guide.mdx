# Migrate an Audience

## Overview 

The API Comparison table highlights the main actions and API methods for creating and managing Audiences. Select an endpoint from IQM or The Trade Desk to review that platform's relevant API documentation. By comparing APIs, you can migrate your workflow from The Trade Desk to IQM.

## Definitions

<center>
<table className='platformTable'>
<th>IQM</th>
<th>The Trade Desk</th>
<th>Description</th>
<tr>
  <td>Audience</td>
  <td>Audience</td>
  <td>The group of people targeted for a Campaign. Audiences comprise included/excluded demographic data which is used to optimize targeting.</td>
</tr>
<tr>
  <td>Audience Segment</td>
  <td>Data Groups</td>
  <td>A group of specified Segments with optimization settings.</td>
</tr>
<tr>
  <td>Segment Group</td>
  <td>Data Elements</td>
  <td>The demographic characteristics by which a group of people can be identified, organized, and combined into Audience Segments (Data Groups for TTD).</td>
</tr>
<tr>
  <td>Matched Audience</td>
  <td>First-Party Data Elements</td>
  <td>A customized Audience that you've created and uploaded to the platform.</td>
</tr>
<tr>
  <td>Pre-bid Audience</td>
  <td>Third-Party Data Elements</td>
  <td>An Audience composed of segments that have been sent to the platform by third-party data providers that you can purchase and target in your Audiences.</td>
</tr>
<tr>
  <td>Geo-Farmed Audience</td>
  <td>Geofence Targeting</td>
  <td>An Audience created based on a customizable geographic location rather than demographic data.</td>
</tr>
<tr>
  <td>Contextual Audience</td>
  <td>Contextual Categories</td>
  <td>An Audience generated by keywords or URLs, based on user activity.</td>
</tr>
</table></center>
