# Migrate a Bid Model

## Overview

The Definitions and API Comparison tables highlight the main terminology, actions, and API methods for managing Bid Modeling. Select an endpoint from IQM or Beeswax to review that platform's relevant API documentation. By comparing APIs, you can migrate your Bid Modeling workflows from Beeswax to IQM.

## Definitions

  <td>Bid Model</td>
  <td>Bid Model</td>
  <td>	The set of optimizations applied to a Campaign or Insertion Order. These optimizations control where ads appear, manage priorities, and dimensions.</td>
</tr>
<tr>
  <td>Bid Model Data</td>
  <td>Terms</td>
  <td>A Bid Model settting that adjusts a Campaign's base bid based on specified parameters.</td>

<tr>
  <td>Dimension</td>
  <td>[Targeting Key<ExternalLink />](https://api-docs.freewheel.tv/beeswax/v2.0/bid-modifiers#supported-targeting-keys)</td>
  <td>A property in digital advertising used to categorize, filter, or analyze audiences, Campaigns or inventories. Serves as a targeting parameter for advertisers, specifying where an ad appears.</td>
</tr>
<tr>
  <td>Bid Modifier</td>
  <td>Bid Modifier</td>
  <td>A multiplier applied to a base bid.</td>
</tr>
<tr>
  <td>Spend Ratio</td>
  <td style={{ textWrap: "nowrap" }}>[Delivery Modifiers<ExternalLink />](https://api-docs.freewheel.tv/beeswax/v2.0/delivery-modifiers)</td>
  <td>A setting that determines how the Campaign budget is distributed across targeting dimensions.</td>
</tr>
<tr>
  <td>Dimension Priority</td>
  <td>Weight</td>
  <td>A setting that dictates the order in which a Campaign should attempt to deliver impressions based on the available budget.</td>
</tr>
