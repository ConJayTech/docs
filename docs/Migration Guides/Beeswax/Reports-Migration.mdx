# Migrate a Report

## Overview

Beeswax uses terminology similar to IQM to create and execute reports.

IQM's [Reports API](/docs/Guidelines/Reports-API-Guidelines) generates insights to inform current and future Campaign optimization. See the [Reporting Solutions<ExternalLink />](https://partner.thetradedesk.com/v3/portal/reds/overview) page for Beeswax's documentation.

For a detailed breakdown of the report creation workflow see the [Run a Report](/docs/Quickstart%20Guides/Reporting-API-Quickstart-Guide) and [Schedule a Report](/docs/Quickstart%20Guides/Schedule-Report-API-Quickstart-Guide) quickstart guides. See the [Reporting<ExternalLink />](https://api-docs.freewheel.tv/beeswax/v2.0/docs/reporting) page on Beeswax's website. 

The table below highlights the main actions and API methods for creating, scheduling, and executing a report. Each IQM endpoint is paired with the analogous API for Beeswax. Both link to their respective documentation. By following the workflow instructions and comparing APIs, the user can migrate their data from Beeswax to IQM.
