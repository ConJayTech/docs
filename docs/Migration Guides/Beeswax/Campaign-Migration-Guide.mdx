# Migrate a Campaign

## Overview

As outlined in the [Overview](/docs/Migration%20Guides/Beeswax/Overview) page, Beeswax refers to their Campaigns as Line Items; and their Insertion Orders as Campaigns. This page foregrounds IQM's usage.

IQM's [Campaign API](/docs/Guidelines/Campaign-API-Guidelines) manages Campaigns and Insertion Orders. See the [Line Items<ExternalLink />](https://api-docs.freewheel.tv/beeswax/v2.0/docs/line-item-configuration) page for Beeswax's platform.  

The table below highlights the main actions and API methods for creating a Campaign. Each IQM endpoint is paired with the analogous API for Beeswax. Both link to their respective documentation. By following the workflow instructions and comparing APIs, the user can migrate their data from Beeswax to IQM. 

 For a detailed breakdown of the Campaign creation workflow see the [Create a Campaign](/docs/Quickstart%20Guides/Upload-Creative-and-Create-a-Campaign-API-Quickstart-Guide) quickstart guide. See the [Line Item Configuration<ExternalLink />](https://api-docs.freewheel.tv/beeswax/v2.0/docs/creating-a-line-item) page on Beeswax's website.
