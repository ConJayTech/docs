# Migrate a Conversion

## Overview

Beeswax refers to their conversions as conversion tags and event tags. This page foregrounds IQM's usage. 

IQM's [Conversions API](/docs/Guidelines/Conversion-API-Guidelines) manages Pixel and Postback conversions. See the [Conversion Events<ExternalLink />](https://api-docs.freewheel.tv/beeswax/docs/conversion-events) page for Beeswax's platform.

<div className="child4">
:::warning

The Conversion Events page for the Beeswax API corresponds to their legacy 0.5 versioning. Information may not accurately reflect the endpoints linked here for the 2.0 versioning. 

See the [Migrating from 0.5 to 2.0 REST API<ExternalLink />](https://api-docs.freewheel.tv/beeswax/v2.0/docs/migrating-from-05-to-20-rest-api) page for more information. 

:::
</div>

The table below highlights the main actions and API methods for managing conversions. Each IQM endpoint is paired with the analogous API for Beeswax. Both link to their respective documentation. By following the workflow instructions and comparing APIs, the user can migrate their data from Beeswax to IQM.
