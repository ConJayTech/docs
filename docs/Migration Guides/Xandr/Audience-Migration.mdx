# Migrate an Audience

## Overview 

The API Comparison table highlights the shared actions and API methods between IQM and Xandr for creating and managing Audiences. Select an endpoint to review each platform's relevant API documentation. By comparing APIs, you can migrate your workflow from Xandr to IQM.

For a comprehensive overview of IQM's Audience services, see the [Audience API](/Guidelines/Audience-API-Guidelines) Guidelines.

## Definitions

<center>
<table className='platformTable'>
<th>IQM</th>
<th>Xandr</th>
<th>Description</th>
<tr>
  <td>Audience</td>
  <td>Profile</td>
  <td>The group of people targeted for a Campaign. Audiences comprise included/excluded demographic data which is used to optimize targeting.</td>
</tr>
<tr>
  <td>Audience Segment</td>
  <td>Segment Groups</td>
  <td>A group of specified Segments with optimization settings.</td>
</tr>
<tr>
  <td>Segment Group</td>
  <td>Segments</td>
  <td>The demographic characteristics by which a group of people can be identified, organized, and combined into Audience Segments (Segment Groups for Xandr).</td>
</tr>
<tr>
  <td>Matched Audience</td>
  <td>Targeting Keys/Values</td>
  <td>A customized Audience that you've created and uploaded to the platform. Xandr uses keys/values to create a customized Audience.</td>
</tr>
<tr>
  <td>Retargeted Audience</td>
  <td>Segment Pixel</td>
  <td>An Audience created by tracking tag Conversions that retargets users who visit the tracked site.</td>
</tr>
</table></center>
